# XUnity Arabic Translator — توثيق المشروع

## ما هو المشروع

Plugin لـ XUnity.AutoTranslator يترجم ألعاب Unity من اليابانية (أو أي لغة) إلى العربية مع **Arabic Reshaping** — تشكيل الحروف العربية بشكلها الصحيح (أول، وسط، آخر، منفصل).

## المشكلة الأساسية

محركات ألعاب Unity ما تدعم العربي صح:
1. **الحروف مفصولة**: الحروف تظهر بشكلها المنفصل بدل المتصل
2. **ترتيب الأسطر مقلوب**: لما النص يلتف بالـ textbox، تقرأه من تحت لفوق بدل من فوق لتحت
3. **الأقواس ما تنعكس**: `()` لازم تصير `)(` للعربي

## البنية

```
plugin/
  ArabicTranslatorPlugin/
    ArabicGoogleTranslate.cs    ← Translator Endpoint (HttpEndpoint)
    ArabicReshaper.cs           ← Arabic reshaping + BiDi + line wrapping
    LineReorderer.cs            ← Harmony hook لترتيب الأسطر بدقة
    ArabicGoogleTranslate.csproj
  libs/
    Developer/                  ← XUnity.AutoTranslator developer DLLs (v5.5.2)
```

## الملفات الرئيسية

### ArabicGoogleTranslate.cs
- يرث من `HttpEndpoint` (XUnity API)
- يرسل النص لـ Google Translate API
- يطبق `ArabicReshaper.Process()` على الترجمة
- يفعّل `LineReorderer.Setup()` عند أول طلب ترجمة (deferred — لازم يتأخر عشان Unity assemblies تكون محمّلة)
- فيه debug log يكتب بـ `arabic_debug.log`

### ArabicReshaper.cs
- **Reshape**: يحول الحروف من Unicode logical لـ Presentation Forms حسب موقعها بالكلمة
- **جدول الحروف**: 28 حرف عربي + فارسي/أوردو (پ چ ژ ک گ ی وغيرها)
- **Lam-Alef ligatures**: لام + ألف = لا (حرف واحد)
- **ApplyBidi**: عكس النص للعرض في محركات LTR + عكس الأقواس + حفظ الأرقام
- **WrapAndReverse**: لف النص عند عرض معين + عكس ترتيب الأسطر
- **Process**: الـ pipeline الكامل (reshape → bidi → optional wrapping)
- يتعامل مع `\n` literal (backslash+n) و actual newlines

### LineReorderer.cs
- يسوي Harmony hook على `UnityEngine.UI.Text.set_text` و `TMPro.TMP_Text.set_text`
- **يستخدم Harmony عبر reflection بالكامل** — ما عنده compile-time dependency
- يدعم Harmony 1.x و 2.x (يكتشف تلقائي)
- بعد ما XUnity يحط النص على الـ text component:
  1. يقرأ `preferredWidth` (عرض النص الفعلي بالبكسل)
  2. يقرأ `rectTransform.rect.width` (عرض الـ textbox بالبكسل)
  3. لو النص أعرض: `charsPerLine = text.Length × (rectWidth / preferredWidth)`
  4. يلف ويعكس ترتيب الأسطر
- يستخدم `_busy` flag لمنع recursion

## المشاكل الي واجهتنا وحلولها

### 1. Harmony Version Mismatch
**المشكلة**: كنا نبني ضد Harmony 2.3.3 (NuGet) بس اللعبة عندها Harmony 1.2.0.1
**الخطأ**: `TypeLoadException: Could not resolve type 'HarmonyLib.Harmony' in assembly '0Harmony, Version=2.3.3.0'`
**الحل**: شلنا الـ NuGet dependency بالكامل واستخدمنا **pure reflection** — نلقى 0Harmony المحمّل بالـ AppDomain ونستخدمه مباشرة. يدعم Harmony 1.x (`Harmony.HarmonyInstance`) و 2.x (`HarmonyLib.Harmony`).

### 2. Hook Timing
**المشكلة**: `LineReorderer.Setup()` بالـ `Initialize()` يفشل لأن Unity UI assemblies ما تكون محمّلة بعد
**الحل**: نأخر الـ Setup لأول `OnExtractTranslation()` call — بهالوقت اللعبة شغّالة وكل الـ assemblies محمّلة.

### 3. ترتيب الأسطر (الأصعب)
**المشكلة**: لما النص العربي (المعكوس للـ LTR) يلتف بالـ textbox، الأسطر تكون بترتيب مقلوب — تقرأ من تحت لفوق.
**الحلول الي جربناها**:
- `MaxLineWidth` ثابت → ما ينفع لأن كل textbox حجم مختلف
- `AutoLineWidth` (تقدير من طول النص الأصلي) → تقريبي وأحياناً غلط
- `RTL Override` (U+202E) → اللعبة ما تدعمه
- **Harmony Hook + `preferredWidth`** ← **الحل الصحيح!** يقرأ العرض الفعلي من الـ component

### 4. الكاش
**المشكلة**: بعد تعديل الـ DLL، النتائج ما تتغير
**السبب**: XUnity يحفظ الترجمات بـ `_AutoGeneratedTranslations.txt` ويحمّلها بدون ما يسأل الـ plugin
**الحل**: حذف ملف الكاش قبل كل اختبار

### 5. مكان الـ DLL
**المشكلة**: الـ DLL لازم يكون بمكانين
**الحل**: نسخه بـ `Managed/` وبـ `Managed/Translators/`

## الإعدادات (XUnity config)

```ini
[Service]
Endpoint=ArabicGoogleTranslate

[General]
Language=ar
FromLanguage=ja

[Behaviour]
MaxCharactersPerTranslation=500
MinDialogueChars=2

[ArabicGoogle]
ServiceUrl=https://translate.googleapis.com/translate_a/single?client=gtx&sl={0}&tl={1}&dt=t&q={2}
AutoLineWidth=True
LineWidthScale=0.9
FixedLineWidth=0
```

## التثبيت (للمستخدم)

1. ثبّت XUnity.AutoTranslator (ReiPatcher أو BepInEx)
2. انسخ `ArabicGoogleTranslate.dll` إلى:
   - `{Game}_Data/Managed/Translators/`
   - `{Game}_Data/Managed/`
3. عدّل الـ config: `Endpoint=ArabicGoogleTranslate` و `Language=ar`

## المشاكل المتبقية

1. **لخبطة بسيطة ببعض النصوص الطويلة**: الـ Hook يحسب `charsPerLine` من نسبة `rectWidth/preferredWidth` × عدد الحروف. هذا دقيق بالمتوسط لكن الحروف العربية عرضها مختلف (مثلاً "من" أضيق من "مختلفة"). أحياناً كلمة تطلع لحالها بسطر. الحل المثالي: حساب عرض كل كلمة بالبكسل عبر `Font.GetCharacterInfo()` بدل عد الحروف.
2. **نصوص ما تنترجم**: بعض النصوص XUnity يتجاهلها (حسب إعداداته)
3. **خط عربي**: بعض الألعاب خطها ما يدعم كل الحروف العربية

## الأدوات المستخدمة

- .NET SDK 9.0 (لبناء الـ DLL)
- Target: net35 (لتوافق Unity)

- لا يحتاج BepInEx — يشتغل مع ReiPatcher
