# التوثيق التقني — XUnity Arabic Translator

## البنية

```
plugin/ArabicTranslatorPlugin/
├── ArabicGoogleTranslate.cs    ← Translator Endpoint (HttpEndpoint)
├── ArabicReshaper.cs           ← Arabic reshaping + BiDi + line wrapping
├── LineReorderer.cs            ← Harmony hook لترتيب الأسطر بدقة
└── ArabicGoogleTranslate.csproj
```

## ArabicGoogleTranslate.cs

يرث من `HttpEndpoint` (XUnity API):
- يرسل النص لـ Google Translate API (`translate.googleapis.com`)
- يطبق `ArabicReshaper.Process()` على الترجمة
- يفعّل `LineReorderer.Setup()` عند أول طلب ترجمة (deferred — لازم يتأخر عشان Unity assemblies تكون محمّلة)
- لو `LineReorderer.IsActive`: ما يلف بنفسه — يخلي الـ Hook يتكفل
- لو الـ Hook ما اشتغل: يلف بالتقدير (AutoLineWidth) كـ fallback
- فيه debug log يكتب بـ `arabic_debug.log`

## ArabicReshaper.cs

### Reshape
يحول الحروف من Unicode logical لـ Presentation Forms حسب موقعها بالكلمة:
- **Isolated** (منفصل): حرف لحاله
- **Initial** (أول): أول حرف بالكلمة
- **Medial** (وسط): حرف بالنص
- **Final** (آخر): آخر حرف

جدول الحروف: 28 حرف عربي + فارسي/أوردو (پ چ ژ ک گ ی وغيرها).

### Lam-Alef Ligatures
لام + ألف = حرف واحد (لا، لأ، لإ، لآ).

### ApplyBidi
عكس النص للعرض في محركات LTR:
- يبني grapheme clusters (حرف + تشكيل)
- يعكس الكل
- يرجع الأرقام واللاتيني لترتيبهم الأصلي
- يعكس الأقواس `()[]{}«»`

### WrapAndReverse
لف النص عند عرض معين + عكس ترتيب الأسطر.

### StripTashkeel
إزالة التشكيل (فتحة ضمة كسرة تنوين) — يحسّن العرض والحساب.

## LineReorderer.cs

**الجزء الأهم** — يسوي Harmony hook على text components:

### كيف يشتغل
1. يبحث عن `0Harmony` المحمّل بالـ AppDomain (أي نسخة)
2. يستخدم **reflection بالكامل** — ما عنده compile-time dependency
3. يدعم Harmony 1.x (`Harmony.HarmonyInstance`) و 2.x (`HarmonyLib.Harmony`)
4. يسوي patch postfix على `Text.set_text` و `TMP_Text.set_text`

### بعد ما XUnity يحط النص:
1. يقرأ `preferredWidth` — عرض النص بالبكسل لو كان سطر واحد
2. يقرأ `rectTransform.rect.width` — عرض الـ textbox بالبكسل
3. لو `preferredWidth > rectWidth`:
   - يحسب `avgCharWidth = preferredWidth / text.Length`
   - يلف كلمة كلمة لما المجموع يتجاوز `rectWidth`
   - يعكس ترتيب الأسطر
4. يستخدم `_busy` flag لمنع recursion

## المشاكل الي واجهتنا وحلولها

### Harmony Version Mismatch
**المشكلة**: بنينا ضد Harmony 2.3.3 بس اللعبة عندها 1.2.0.1
**الحل**: شلنا NuGet dependency واستخدمنا pure reflection

### Hook Timing
**المشكلة**: `Setup()` بالـ `Initialize()` يفشل — UI assemblies ما تكون محمّلة
**الحل**: أخّرنا الـ Setup لأول `OnExtractTranslation()`

### ترتيب الأسطر
**جربنا**: FixedLineWidth → AutoLineWidth → RTL Override → Harmony Hook
**الحل النهائي**: Harmony Hook + `preferredWidth`

### الكاش
**المشكلة**: النتائج ما تتغير بعد تعديل الـ DLL
**الحل**: حذف `_AutoGeneratedTranslations.txt`

### مكان الـ DLL
**المشكلة**: لازم يكون بمكانين
**الحل**: `Managed/` و `Managed/Translators/`

## الإعدادات

```ini
[ArabicGoogle]
ServiceUrl=...                  # Google Translate API URL
StripTashkeel=True              # إزالة التشكيل
AutoLineWidth=True              # تقدير عرض السطر (fallback)
LineWidthScale=0.9              # معامل عرض السطر للـ fallback
FixedLineWidth=0                # عرض ثابت (0 = معطّل)
```

## المشاكل المتبقية

1. **لخبطة بسيطة ببعض النصوص الطويلة**: `avgCharWidth` متوسط — الحروف العربية عرضها مختلف. الحل المثالي: `Font.GetCharacterInfo()` لكل حرف.
2. **نصوص ما تنترجم**: بعض النصوص XUnity يتجاهلها حسب إعداداته.
